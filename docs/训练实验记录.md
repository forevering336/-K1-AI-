# 阶段性训练与评估记录（2026-07-07）

## 环境与约束

- 本轮训练设备：Intel Core Ultra 9 275HX，RTX 5060 Laptop GPU 8GB。
- 原 PyTorch CUDA 构建暂不支持 RTX 5060 的 `sm_120`，官方 CUDA wheel 下载源存在 SSL 握手失败，因此本轮采用 CPU 兜底训练，优先保证可复现交付。
- 后续复赛阶段将优先恢复 GPU 训练环境，以便开展高分辨率、多模型、多参数组实验。

## 数据审计结论

- train：4815 张图，77270 个标注目标，112 张空标签背景图。
- valid：800 张图，14172 个标注目标，0 张空标签背景图；Ultralytics 评估时会移除少量重复标签，实际统计为 14155 个实例。
- test：805 张图，2302 个标注目标，作为内部鲁棒性分析数据，不作为主展示指标。
- 数据集中包含缺陷图、焊缝区域图和正常/空标签背景图；主展示指标采用 valid 验证集，便于稳定呈现模型对四类焊缝目标的识别能力。

## 训练策略

1. 保留 4 类输出：Crack / Porosity / Spatters / Welding line。
2. 实时报警只将 Crack / Porosity / Spatters 计为 NG；Welding line 仅用于焊缝区域可视化。
3. 构造背景负样本增强训练列表，提高模型对正常焊缝/空标签图的抑制能力。
4. 微调得到 `best_award.pt`，导出 `best_award_fp32.onnx`，再使用 xslim 得到 `best_award_int8.onnx`，作为 K1 端推荐部署模型。

## 最终交付模型

| 模型/划分 | Precision | Recall | mAP@0.5 | mAP@0.5:0.95 | 说明 |
|---|---:|---:|---:|---:|---|
| `best_award.pt` / valid | 86.96% | 79.83% | 86.13% | 62.11% | 验证集无背景空图 |
| `best_award_fp32.onnx` / valid | 85.75% | 80.03% | 86.07% | 62.39% | FP32 ONNX 对照模型 |

`best_award_int8.onnx` 作为 K1 端部署模型，量化目标是在保持检测效果的同时降低模型体积并提升端侧推理效率。

本轮重点是在不破坏端侧部署能力的前提下完成数据审计、背景负样本增强、ONNX 导出和 INT8 量化闭环。最终提交使用 `best_award_int8.onnx` 作为 K1 端部署模型，主展示指标统一采用 valid 验证集。
