# 源码说明

本交付包是给制作报告同学使用的**最终统一版本**，当前推荐方案为：USB 摄像头实时检测 + Web 看板 + 声光报警 + PatchCore 未知异常分支 + award INT8 部署模型。

## 入口文件

- `weld_live.py`：主程序入口，负责 USB 摄像头采集、YOLO 推理、画框、Web 视频流、报警联动和截图保存。
- `start.sh`：K1 板端一键启动脚本，默认加载 `models/best_award_int8.onnx`。

## 核心源码

| 文件 | 作用 |
|---|---|
| `weld_live.py` | 实时检测主程序 |
| `web_server.py` | Flask Web 仪表盘后端 |
| `web/templates/index.html` | 离线 Web 前端页面 |
| `usb_alarm.py` | USB 继电器红绿灯控制 |
| `alarm.py` | GPIO/软件报警封装备用 |
| `anomaly_scorer.py` | PatchCore 未知异常检测分支 |
| `eval_map.py` | ONNX mAP 评估脚本 |
| `test_cam.py` | 摄像头测试工具 |
| `light_test.py` | 报警灯测试工具 |

## 模型文件

| 文件 | 说明 |
|---|---|
| `models/best_award_int8.onnx` | 推荐 K1 端 INT8 部署模型 |
| `models/best_award_fp32.onnx` | FP32 对照模型 |
| `models/best_award.pt` | 对应训练权重 |
| `models/backbone3.onnx` | PatchCore 特征提取 backbone |
| `models/memory_bank2.npy` | PatchCore 正常样本记忆库 |

## K1 运行方式

进入交付包目录后执行：

```bash
chmod +x start.sh
./start.sh
```

或手动运行：

```bash
python3 weld_live.py --web-video --no-display --model models/best_award_int8.onnx
```

浏览器访问：

```text
http://<K1的IP>:5000
```

## PC 端评估方式

```bash
yolo detect val model=models/best_award_fp32.onnx data=data_weld_local.yaml split=val imgsz=640 batch=16 device=cpu plots=True
yolo detect val model=models/best_award_int8.onnx data=data_weld_local.yaml split=val imgsz=640 batch=16 device=cpu plots=True
```

对应日志位于：

- `eval/` 中的验证集评估日志

评估图表位于：

- `eval/best_award_fp32_valid_assets/`

## 报警逻辑

系统输出 4 类目标，但只将以下 3 类作为 NG 缺陷：

- Crack
- Porosity
- Spatters

`Welding line` 仅用于显示焊缝区域，不触发 NG 报警。

## 依赖

依赖见 `requirements.txt`。K1 板端建议使用系统 Python 环境，并确保安装：

```bash
pip3 install flask numpy opencv-python onnxruntime pyyaml
```

如 K1 环境支持 `spacemit_ort`，程序会优先使用 SpaceMITExecutionProvider；否则自动退回 CPUExecutionProvider。
