# 项目总结

## 项目名称

基于 RISC-V K1 边缘 AI 平台的工业焊接缺陷智能检测系统

## 项目定位

本项目面向工业焊接质检场景，基于进迭时空 K1 RISC-V 边缘计算平台实现一套端侧焊缝缺陷检测系统。系统不是单一图片识别脚本，而是一个完整的边缘 AI 质检终端，覆盖图像采集、模型推理、缺陷判定、声光报警、Web 远程监控、日志记录和缺陷截图留存。

## 核心功能

1. 焊缝图像实时采集：使用 USB 摄像头采集焊缝图像，适配 K1 Linux 环境。
2. 已知缺陷检测：YOLOv8 检测裂纹、气孔、飞溅，并显示焊缝区域。
3. 未知异常兜底：PatchCore 分支用于发现训练集之外的异常缺陷。
4. INT8 边缘部署：使用 xslim 将 ONNX 模型量化为 INT8，适配 K1 NPU 推理。
5. 声光报警闭环：检测到 NG 缺陷后触发 USB 继电器红灯报警，OK 状态显示绿灯。
6. Web 远程看板：局域网浏览器查看实时画面、FPS、缺陷统计、历史记录和截图。
7. 本地数据留痕：检测日志保存为 CSV，缺陷画面自动截图，便于后续追溯。

## 技术路线

图像输入经过 letterbox、归一化和 NCHW 排布后送入 YOLOv8 ONNX 模型。模型输出经置信度筛选、坐标还原和 NMS 后得到目标框。系统将 Crack、Porosity、Spatters 判定为缺陷类别，触发 NG；Welding line 作为焊缝区域提示框显示，不触发报警。PatchCore 分支与 YOLO 并联，对未知异常进行补充检测。

模型部署流程为：

```text
best_award.pt
  -> best_award_fp32.onnx
  -> xslim INT8 静态量化
  -> best_award_int8.onnx
  -> K1 ONNX Runtime / spacemit_ort 推理
```

## 数据与结果

本项目整理的数据集包含 6420 张焊缝图像，约 9.37 万个标注目标，覆盖裂纹、气孔、飞溅、焊缝区域和正常背景样本。

当前推荐部署模型为 `best_award_int8.onnx`，主展示指标采用 valid 验证集：

| 模型/划分 | Precision | Recall | mAP@0.5 | mAP@0.5:0.95 | 说明 |
|---|---:|---:|---:|---:|---|
| best_award.pt / valid | 86.96% | 79.83% | 86.13% | 62.11% | PyTorch 权重 |
| best_award_fp32.onnx / valid | 85.75% | 80.03% | 86.07% | 62.39% | FP32 ONNX 对照模型 |

INT8 模型相比 FP32 保持同级别检测表现，同时模型体积更小，更适合 K1 端部署。

## 创新点

1. 已知缺陷检测与未知异常检测双分支融合，提高工业现场适应能力。
2. 焊缝区域显示与缺陷报警解耦，画面更专业，判定逻辑更符合质检语义。
3. 使用 xslim INT8 静态量化适配 K1 NPU，兼顾模型体积、速度和精度保持。
4. 完整实现边缘端闭环：检测、报警、Web、日志、截图均在本地完成。
5. 前端无 CDN、系统无云端依赖，适合断网或内网工业环境。
